I2 je rozhraní mezi MSC Serverem pro přepojování okruhů rozšířeným pro služby IMS Centralized Services a domácím IMS, které umožňuje poskytování služeb IMS starším zařízením připojeným přes CS tím, že umožňuje MSC vystupovat jako aplikační server IMS.

## Popis

Referenční bod I2 je kritické rozhraní v architektuře služeb IMS Centralized Services (ICS) dle 3GPP. Je definováno mezi Mobile Switching Centre ([MSC](/mobilnisite/slovnik/msc/)) Serverem, který byl rozšířen o schopnosti ICS (často nazývaným ICS-enhanced MSC Server nebo MSC Server [AS](/mobilnisite/slovnik/as/)), a domácí sítí IMS, konkrétně jádrem IMS zahrnujícím prvky jako Serving-Call Session Control Function (S-CSCF). Jeho primární funkcí je umožnit uživateli připojenému přes starší rádiový přístup s přepojováním okruhů ([CS](/mobilnisite/slovnik/cs/)) (např. [GERAN](/mobilnisite/slovnik/geran/), UTRAN) přístup a využívání služeb hostovaných v IP doméně IMS. Rozhraní I2 umožňuje MSC Serveru chovat se vůči jádru IMS jako aplikační server IMS (AS).

Z architektonického hlediska obsahuje ICS-enhanced MSC Server interní funkci IMS Service Centralization and Continuity (SCC) a funkci aplikačního serveru IMS. Když připojené UE (přes CS) iniciuje nebo přijímá relaci, MSC Server používá referenční bod I2 pro interakci s domácí sítí IMS. Pro volání iniciované mobilním zařízením ([MO](/mobilnisite/slovnik/mo/)) využívá MSC Server I2 k provedení registrace v IMS jménem uživatele (pokud již nebyla provedena) a k směrování signalizace relace (např. SIP INVITE) do jádra IMS. S-CSCF v IMS pak požadavek zpracuje, aplikuje servisní logiku z jiných AS (jako je Telephony Application Server) a volání směruje. Pro volání přijímané mobilním zařízením ([MT](/mobilnisite/slovnik/mt/)) směrované k uživateli ICS směruje IMS SIP relaci přes rozhraní I2 na MSC Server AS, který následně převede signalizaci SIP na CS signalizaci (např. [ISUP](/mobilnisite/slovnik/isup/)), aby doručil volání k UE přes CS doménu.

Rozhraní I2 je v zásadě rozhraní založené na SIP, typicky používající pro ověřování účastníka a načítání profilu rozhraní Cx založené na Diameter (mezi MSC Server AS a Home Subscriber Server ([HSS](/mobilnisite/slovnik/hss/))). Umožňuje konzistenci služeb, což uživateli poskytuje stejné služby založené na IMS (jako je kontinuita hlasového volání, doplňkové služby) bez ohledu na to, zda je připojen přes CS nebo PS přístup. MSC Server používá I2 k hlášení stavu připojení uživatele přes CS do IMS, což je klíčové pro rozhodování o směrování ukončovaných volání. Rozhraní také podporuje schopnosti jako zrušení registrace iniciované sítí a aktualizace registrace. Tím, že poskytuje tento most, je I2 klíčovým prvkem migrační strategie ze sítí CS na plně IP sítě IMS, což operátorům umožňuje zavádět a centralizovat služby v IMS a zároveň stále podporovat velkou základnu starších telefonů pouze s podporou CS a oblasti pokrytí CS rádiovým signálem.

## K čemu slouží

Rozhraní I2 bylo vytvořeno, aby vyřešilo hlavní výzvu přechodu při evoluci od telefonie s přepojováním okruhů k IP multimediálnímu subsystému (IMS). Před ICS mohli uživatelé připojení přes CS doménu přistupovat pouze k tradičním CS službám, zatímco služby IMS byly výhradní pro uživatele připojené přes přístup s přepojováním paketů (PS), jako je LTE. To vytvářelo služební dichotomii a bránilo nasazení nových služeb založených na IMS, protože operátoři museli udržovat duplicitní servisní platformy pro uživatele CS a PS. Služby IMS Centralized Services (ICS) byly koncipovány tak, aby centralizovaly veškerou servisní logiku a její provádění v IMS, bez ohledu na typ přístupu uživatele.

Referenční bod I2 je technickou realizací, která umožňuje tuto vizi pro uživatele připojené přes CS. Řeší omezení starší sítě CS, která není schopna přímo komunikovat s jádrem IMS založeným na SIP. Definováním standardizovaného rozhraní, kde může MSC Server vystupovat jako aplikační server IMS, se CS síť stává pouhou přístupovou větví k servisní platformě IMS. To umožňuje operátorům vyřadit starší CS servisní platformy (jako tradiční IN systémy) a rychle nasazovat nové služby v IMS, čímž jsou okamžitě dostupné uživatelům CS i PS. Bylo to strategické řešení pro urychlení transformace sítě, snížení provozních nákladů a poskytnutí jednotné uživatelské zkušenosti, což vedlo k vytvoření a standardizaci rozhraní I2 ve verzi 3GPP Release 12.

## Klíčové vlastnosti

- Umožňuje ICS-enhanced MSC Serveru vystupovat jako aplikační server IMS (AS)
- Rozhraní založené na SIP mezi prvkem CS sítě a domácím jádrem IMS (S-CSCF)
- Podporuje registraci v IMS a iniciaci relace jménem UE připojených přes CS
- Umožňuje směrování IMS relací přijímaných mobilním zařízením (MT) k uživatelům připojeným přes CS
- Umožňuje konzistenci služeb napříč přístupovými doménami CS a PS
- Klíčová komponenta architektury služeb IMS Centralized Services (ICS)
