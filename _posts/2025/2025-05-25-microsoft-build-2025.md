Microsoft na své výroční konferenci Build 2025, konané ve dnech 19. - 22. května, představil více než 50 novinek zaměřených na vývojářskou komunitu. Hlavním tématem letošního ročníku je umělá inteligence, především pak agentníAI, která má zefektivnit práci vývojářů a transformovat způsob vývoje softwaru. Pojďme se podívat na nejdůležitější oznámení.

### Microsoft 365 Copilot

Microsoft rozšiřuje své AI řešení v rámci Microsoft 365 Copilot o nové funkce. Tzv. "Wave 2" jarní vydání přináší aktualizovanou aplikaci Microsoft 365 Copilot navrhovanou pro lepší spolupráci mezi člověkem a AI agentem. Součástí je nová funkce "Create", která využívá generativní schopnosti modelu OpenAI GPT-4o pro tvorbu obrázků. 

Copilot Notebooks umožní přeměnit obsah a data na okamžité postřehy a akce. Funkce Copilot Search a Copilot Memory budou uvedeny v červnu. Specializovaní agenti Researcher a Analyst, první svého druhu pro pracovní prostředí, budou celosvětově dostupní tento měsíc prostřednictvím programu Frontier.

V rámci Outlooku přináší Copilot nové funkce, které usnadňují navigaci v e-mailech poskytováním souhrnů výsledků vyhledávání a přiložených souborů přímo v e-mailech. Pomáhá také uživatelům lépe se připravit na schůzky poskytováním relevantního kontextu, úkolů a dokumentů souvisejících s tématem schůzky.

### Copilot Tuning

S Microsoft 365 Copilot Tuning mohou zákazníci využít vlastní firemní znalosti k trénování modelů, které provádějí úlohy specifické pro danou doménu s vyšší rychlostí a přesností. Copilot Tuning zajišťuje, že přístup k naladěnému modelu je omezen na oprávnění základních tréninkových dat v rámci podniku.

Agenti vytvořeni pomocí Microsoft 365 Copilot's Agent Builder mohou tyto naladěné modely využívat. Například právnická firma může vytvořit agenta, který generuje dokumenty v souladu se stylem, strukturou a odbornými znalostmi předchozí práce. Poradenská společnost pracující ve specializovaném odvětví, jako je letectví, může vytvořit Q&A agenta pomocí modelu naladěného na interní dokumenty zabývající se regionálními nebo mezinárodními regulačními požadavky.

### Microsoft Teams a agenti

Microsoft Teams přináší pokročilé vývojářské nástroje, které zahrnují podporu protokolu Agent2Agent (A2A), aktualizovanou knihovnu Teams AI a agentní paměť. Tyto prvky umožní vytvářet chytřejší, spolupracující agenty ke zvýšení produktivity a podnikové efektivity.

Vývojáři budou moci vytvářet agenty pro Teams pomocí protokolu A2A, který umožňuje bezpečnější komunikaci peer-to-peer mezi agenty. Tito agenti si budou vyměňovat zprávy, data a přihlašovací údaje bez spoléhání se na centralizované zprostředkovatele.

## Microsoft Copilot Studio

Microsoft Copilot Studio a Microsoft 365 Copilot se rozšiřují o nové funkce, které usnadňují vývojářům vytvářet výkonnější agenty:

- Microsoft 365 Agents Toolkit pro Visual Studio zjednodušuje vývoj podnikových agentů integrací AI nástrojů.
- Microsoft 365 Agents SDK umožňuje vytvářet podnikové, škálovatelné, vícekanálové agenty.
- Microsoft 365 Copilot APIs je sada API připravených pro podniky, která umožňuje vývojářům vytvářet rychlé, kontextově informované a bezpečné AI zážitky s daty Microsoft 365.
- Agent Store nabízí agenty od Microsoftu, partnerů a zákaznických organizací s personalizovaným objevováním.

Nové funkce zahrnují také podporu pro vícenásobné agentní systémy, což umožňuje více agentům spolupracovat jako tým. Agenti budou moci využívat Computer Use Tool k provádění podnikových úkolů na uživatelských rozhraních, ať už na desktopových či webových aplikacích. Code Interpreter umožní agentům psát a spouštět Python kód k provádění složitých úkolů.

## Azure AI Foundry

Azure AI Foundry je jednotná platforma pro návrh, přizpůsobení a správu AI aplikací a agentů. Rozšiřuje se o nové přímé nabídky, včetně modelů Grok 3 a Grok 3 mini od xAI. Katalog Microsoft nyní zahrnuje přes 1900 modelů.

### Azure AI Foundry Agent Service

Tato služba, nyní obecně dostupná, umožňuje profesionálním vývojářům navrhovat, nasazovat a škálovat podnikové AI agenty pro automatizaci obchodních procesů. S podporou pracovních postupů s více agenty mohou vývojáři orchestrovat specializované agenty pro zpracování složitých úkolů a zvýšení operační efektivity.

Služba zjednodušuje vývoj agentů snadnou integrací se zdroji znalostí, jako jsou Microsoft Bing, SharePoint, Azure Databricks a Microsoft Fabric. Také se připojuje k centralizovanému katalogu nástrojů a předem připravených šablon agentů, které vývojáři mohou snadno přizpůsobit.

### Azure AI Foundry Observability

Azure AI Foundry Observability zavádí nové funkce pro integrované sledování metrik výkonu, kvality, nákladů a bezpečnosti. Tyto schopnosti poskytují hlubší vhled do kvality, výkonu a bezpečnosti agentů.

Nabízí podporu vývojářům během jejich cesty, umožňující hodnocení během ladění modelu, upgrady systémových výzev a přechody mezi modely. Cesta začíná v prostředí pro agenty, kde zákazníci mohou své agenty hodnotit v kontrolovaném prostředí.

### Azure AI Foundry Local

Azure AI Foundry Local bude dostupný na Windows 11 a MacOS a bude zahrnovat inferenci modelu, modely a agenty jako službu a playground pro rychlý a efektivní lokální vývoj AI. Přinese sílu open-source modelů v Azure AI Foundry na klientská zařízení.

### Security v Azure AI Foundry

Azure AI Foundry nyní integruje několik významně vylepšených bezpečnostních funkcí, včetně Prompt Shields, které poskytují ochranu proti útokům jailbreaks a injection. Spotlighting bude jedinečně identifikovat nepřátelské výzvy vložené do externích zdrojů dat oddělením důvěryhodných od nedůvěryhodných vstupů.

## GitHub a AI

### GitHub Copilot Coding Agent

GitHub Copilot se vyvíjí z asistenta v editoru na agentního AI partnera s prvním autonomním, asynchronním vývojářským agentem integrovaným do platformy GitHub. GitHub Copilot bude schopen testovat, iterovat a zdokonalovat kód v GitHubu jako [AI agent](/ai/agenti/), kterému mohou vývojáři delegovat jak rutinní, tak specializované úkoly.

### GitHub Models

GitHub Models poskytne týmům jednotné a důvěryhodné centrum pro průzkum nejlepších modelů a vytváření, ukládání, hodnocení a sdílení promptů, aniž by opustili GitHub. Centralizací hodnocení modelů a promptů v intuitivním prostoru budou uživatelé moci vytvářet, testovat a spravovat AI funkce přímo ze svého repozitáře.

### Open-source GitHub Copilot Chat v VS Code

Microsoft otevírá zdrojový kód GitHub Copilot Chat v VS Code. V průběhu několika následujících měsíců budou AI schopnosti z rozšíření GitHub Copilot součástí open-source repozitáře VS Code. Tento krok odráží závazek k transparentnosti, inovacím řízeným komunitou a poskytnutí vývojářům většího hlasu při utváření budoucnosti vývoje s pomocí AI.

## Windows a AI

### Windows AI Foundry

Windows AI Foundry, evoluce Windows Copilot Runtime, nabízí jednotnou a spolehlivou platformu podporující vývojový cyklus AI od výběru modelu, optimalizace, fine-tuningu a nasazení napříč klientem a cloudem. Zahrnuje několik funkcí:

- Windows ML je základem platformy Windows AI a vestavěným runtime pro AI inference ve Windows.
- Windows AI Foundry integruje Foundry Local a další katalogy modelů jako Ollama a NVIDIA NIMs.
- Windows AI APIs nabízí rychlou a snadnou cestu pro vývojáře aplikací Windows k integraci AI funkcí, které běží lokálně na Copilot+ PC.

### [Model Context Protocol](/ai/mcp/) ([MCP](/ai/mcp/)) na Windows

[MCP](/ai/mcp/) na Windows nabídne standardizovaný rámec pro propojení AI agentů s nativními aplikacemi Windows. Aplikace Windows mohou vystavit specifické funkce ke zvýšení dovedností a schopností agentů nainstalovaných lokálně na Windows PC.

### Bezpečnost ve Windows

Dvě důležité změny v oblasti bezpečnosti Windows pomohou udržet uživatele pod kontrolou jejich bezpečnosti a soukromí:

- Ochrana administrátora je nová bezpečnostní funkce platformy, která zabrání neúmyslnému zvýšení oprávnění zajištěním, že uživatelé se ověří pomocí Windows Hello.
- Přístup k citlivým zdrojům, jako jsou kamera, mikrofon a poloha (C/M/L), bude brzy vyžadovat výslovný souhlas uživatele.

## Databáze a AI

### Microsoft Fabric a Cosmos DB

Microsoft rozšiřuje integraci dat a AI platforem. Rozhovor s daty umožní týmům prozkoumávat a uvažovat nad komplexními datovými sadami jednoduše kladením otázek. Data agenti v Microsoft Fabric jsou AI asistenti, kteří jdou nad rámec jednoduchého získávání dat z OneLake - vedou také přirozené konverzace o nich.

Cosmos DB (NoSQL) v Fabric přinese podnikovou dynamickou škálovatelnost, konzistentní spolehlivost a nízkou latenci pro poskytování polostrukturovaných dat. Vývojáři budou moci nasadit Cosmos DB (NoSQL) několika kliknutími a snadno vytvářet vysoce výkonné, distribuované aplikace.

### SQL Server 2025

Microsoft SQL Server 2025, postavený na základech známých pro špičkovou bezpečnost a výkon, umožní zákazníkům vyvíjet moderní AI aplikace pomocí jejich dat. Poskytne vestavěné, rozšiřitelné AI schopnosti, zvýšenou produktivitu vývojářů a snadnou integraci s Microsoft Azure a Microsoft Fabric - to vše v rámci SQL Server engine pomocí známého jazyka T-SQL.

## Bezpečnost pro AI

### Microsoft Purview SDK

Microsoft Purview SDK poskytne REST API, dokumentaci a ukázky kódu pro začlenění zabezpečení dat a dodržování předpisů Microsoft Purview přímo do AI aplikací z jakéhokoli integrovaného vývojového prostředí.

Začleněním Microsoft Purview REST API do AI aplikací bude kód odesílat data související s výzvami a odpověďmi do Microsoft Purview, který automaticky poskytuje signály týmům pro bezpečnost a dodržování předpisů na podporu vyšetřování.

### Microsoft Entra Agent ID

Microsoft zavádí Microsoft Entra Agent ID, které rozšiřuje jeho řešení identity a přístupu na AI agenty. Řeší problém proliferace AI agentů přiřazením jedinečného identifikátoru každému agentovi v prostředí. S Entra Agent ID budou správci moci vidět všechny AI agenty vytvořené pomocí Copilot Studio a Azure AI Foundry na jednom místě.

## Podpora agentního webu

### [Model Context Protocol](/ai/mcp/) ([MCP](/ai/mcp/))

Microsoft a GitHub se připojili k řídícímu výboru [Model Context Protocol](/ai/mcp/) ([MCP](/ai/mcp/)) a poskytují širokou podporu první strany pro [MCP](/ai/mcp/) napříč svou agentní platformou a frameworky.

Microsoft a GitHub oznamují dva nové příspěvky do ekosystému MCP:

- Specifikace identity a autorizace: Týmy identity a bezpečnosti společnosti Microsoft spolupracovaly s Anthropic, řídícím výborem MCP a širší komunitou MCP na navržení aktualizované autorizační specifikace.
- Veřejný, komunitou řízený registr MCP serverů: GitHub a řídící výbor MCP spolupracovali na návrhu registrační služby pro MCP servery.

### NLWeb

[NLWeb](https://github.com/microsoft/nlweb), nový otevřený projekt, hraje podobnou roli jako HTML pro agentní web. NLWeb usnadňuje webovým stránkám poskytovat konverzační rozhraní pro uživatele s modelem dle jejich výběru a s jejich vlastními daty. To umožňuje uživatelům interagovat přímo s webovým obsahem bohatým, sémantickým způsobem.
