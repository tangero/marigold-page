Anthropic pokračuje ve vývoji svého příkazového nástroje Claude Code, který umožňuje vývojářům pracovat s AI asistentem Claude přímo z terminálu. Poslední čtyři aktualizace (verze 1.0.61 až 1.0.64) přinášejí významná vylepšení v oblasti vlastních agentů, systémových hooks a podpory různých platforem.

Claude Code funguje jako most mezi vývojáři a AI modelem Claude, umožňuje delegovat programátorské úkoly, automatizovat workflow a integrovat AI asistenta do vývojového procesu. Nástroj podporuje vytváření vlastních agentů pro specifické úkoly, hooks pro automatizaci a SDK pro pokročilé integrace.

## Vlastní agenti získávají flexibilitu modelu

Verze 1.0.64 zavedla klíčovou funkcionalitu - možnost specifikovat, který AI model má konkrétní agent používat. Tato změna umožňuje vývojářům optimalizovat výkon jednotlivých agentů podle typu úkolů. Rychlejší modely lze nasadit pro jednodušší úkoly, zatímco složitější operace mohou využívat pokročilejší varianty Claude.

Současně byla opravena chyba, která umožňovala nechtěný přístup k rekurzivnímu nástroju agentů. Tato oprava zvyšuje bezpečnost a stabilitu při práci s více agenty současně.

Praktické využití spočívá v možnosti vytvořit specializované agenty - například jeden pro code review s rychlým modelem, druhý pro komplexní architektonické rozhodnutí s pokročilejším modelem.

## @-mention systém zjednodušuje vyvolání agentů

Verze 1.0.62 přidala [@-mention podporu s automatickým doplňováním](https://docs.anthropic.com/en/docs/claude-code) pro vlastní agenty. Vývojáři nyní mohou agenta vyvolat jednoduchým zápisem `@<název-agenta>`, přičemž systém nabídne dostupné možnosti prostřednictvím typeahead funkcionality.

Tato změna výrazně zrychluje workflow, protože eliminuje nutnost pamatovat si přesné názvy agentů nebo procházet seznamy dostupných nástrojů. Integrace je navržena podobně jako @-mentions v moderních komunikačních nástrojích.

## Windows platforma získala plnou funkcionalnost

Verze 1.0.63 se zaměřila na opravu kritických problémů na Windows platformě. Byla obnovena funkcionalité vyhledávání souborů, @-mentions pro agenty a vlastní slash příkazy, které na této platformě nefungovaly správně.

Tato oprava je významná pro širší adopci nástroje, protože Windows představuje majoritní platformu pro mnoho vývojářských týmů. Předchozí omezení mohla bránit některým organizacím v nasazení Claude Code do jejich workflow.

## Hooks systém rozšiřuje automatizační možnosti

Hooks představují systém pro automatické spouštění akcí v určitých momentech práce s Claude Code. Verze 1.0.62 přidala [SessionStart hook](https://docs.anthropic.com/en/docs/claude-code), který se aktivuje při zahájení nové relace. Tato funkcionalita umožňuje automatické nastavení prostředí, načtení kontextu projektu nebo inicializaci specifických nástrojů.

Současně byl do hooks systému přidán systemMessage field v JSON výstupu, který slouží k zobrazování varování a kontextových informací. Tato změna zvyšuje transparentnost automatizovaných procesů a pomáhá vývojářům pochopit, co se v pozadí děje.

Praktické využití SessionStart hook zahrnuje automatické načtení projektové dokumentace, nastavení coding standards nebo inicializaci specifických vývojových nástrojů pro daný projekt.

## SDK a uživatelské rozhraní

Claude Code SDK v verzi 1.0.64 opravilo problém se sledováním uživatelského vstupu napříč konverzacemi s více koly. Tato oprava je kritická pro aplikace, které využívají SDK pro pokročilé integrace a potřebují udržovat konzistentní stav konverzace.

Verze 1.0.61 přinesla několik vylepšení uživatelského rozhraní. Transcript mode (aktivovaný Ctrl+R) nyní používá Esc klávasu pro ukončení místo přerušení procesu. Tato změna činí ovládání intuitivnějším a snižuje riziko nechtěného přerušení práce.

## Nastavení a integrace s vývojovými prostředími

Nová funkcionalita `--settings` flag umožňuje načítání konfigurace z JSON souboru, což zjednodušuje správu nastavení ve týmových prostředích. Současně byla opravena resoluce cest k nastavením, které jsou symlinky.

Integrace s IDE získala několik vylepšení. VSCode na macOS nyní podporuje vkládání obrázků pomocí ⌘+V. Pro organizace, které preferují manuální připojení, byla přidána možnost `CLAUDE_CODE_AUTO_CONNECT_IDE=false` pro vypnutí automatického připojování k IDE.

Přidaná proměnná prostředí `CLAUDE_CODE_SHELL_PREFIX` umožňuje obalování příkazů spouštěných Claude Code, což poskytuje dodatečnou kontrolu nad prováděnými operacemi.

Komanda `/add-dir` nyní podporuje automatické doplňování cest k adresářům, což zrychluje navigaci ve složitějších projektových strukturách.

## Technické detaily a telemetrie

Verze 1.0.61 opravila problém s OTEL (OpenTelemetry) reportováním nesprávné organizace po změnách autentifikace. Tato oprava je důležitá pro organizace využívající telemetrická data pro monitoring používání nástroje.

Byla také zlepšena spolehlivost kontroly síťového připojení, což snižuje falešné alarmy o problémech s konektivitou.

Bash nástroj získal opravu kontroly oprávnění pro allowed-tools, což zvyšuje bezpečnost při práci se systémovými příkazy.

Tyto aktualizace ukazují, že Anthropic pokračuje v systematickém vývoji Claude Code směrem k profesionálnímu nástroju pro týmovou prácu. Důraz na Windows podporu, rozšiřování automatizačních možností a zlepšování integrace s existujícími vývojovými workflow naznačuje ambici konkurovat etablovaným nástrojům pro AI-asistovaný vývoj.
